정적 리소스란?
보통 특정 css, html, 파일들 등을 두면 서버들이 이 파일들을 서빙(서버) 해준다.
http 요청해주고 http 프로토콜로 응답해준다. 서로 웹서버로 데이터를 주고받게해줌.


WAS (웹 애플리케이션 서버)
큰차이론 프로그램 코드를 실행해서 애플리케이션 로직 수행
http 요청에 따라 사용자마다 다른화면(아이드를 보여준다던가) 해줄수 있다.

was 하나라면??
was는 너무 많은 역할을 하는데 애플리케이션 로직 실행, 정적인 컨텐츠들도 계속 제공하며
서버 과부화 걸릴수 있음.

애플리케이션 로직은 비싸다?
주문 하는 애플리케이션 로직이 있다고 하자 이게 비쌀까? 이미지 하나만 보여주는게 비쌀까?
정적인 파일은 파일하나 땅 불러주기만 하면 끝이다.
애플리케이션 로직은 비지니스 로직이 들어가니 정말 비싸다 디비도 타고 로직도 타고...


그래서 보통은

웹서버를 먼저 두고 (정적) 리소스 먼저 다 처리하고 동적 로직이 필요하면 was에서 처리해서 db 처리한다.
이렇게 하면 was는 정말 중요한 로직만 담당한다.
